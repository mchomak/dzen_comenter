# Команда повторной авторизации Дзена

## Цель

Добавить Telegram-команду `/auth`, которая запускает авторизацию Дзена заново и отправляет пользователю существующее приглашение:

> Дзену нужна авторизация. Готов сейчас войти? Нажми «Готов», и я открою страницу входа.

## Архитектура

### TelegramAuthAssistant

- Добавить неблокирующий метод `poll_auth_command()`.
- Метод получает накопленные Telegram updates с коротким polling-запросом и возвращает `True`, только если найдено точное сообщение `/auth` из настроенного `chat_id`.
- Поддержать `/auth@имя_бота`, чтобы команда работала в групповых чатах без принятия команд от других чатов.
- Добавить метод сброса готовности, который позволяет `ask_ready()` отправить приглашение повторно.
- Сохранить общий offset Telegram updates внутри assistant, чтобы уже обработанные updates не обрабатывались повторно.

### PlaywrightSessionManager

- Добавить `reset_authentication()`.
- Метод очищает cookies текущего persistent-контекста, удаляет локальный файл `STORAGE_STATE_PATH`, открывает страницу комментариев и оставляет браузер готовым к новому login-flow.
- Ошибка удаления отсутствующего файла не считается ошибкой.

### OrchestratorLoop

- В начале каждого `run_cycle()` проверять `/auth`.
- При найденной команде вызвать сброс сессии и состояния приглашения, затем продолжить обычный `_ensure_session()` в том же цикле.
- При отсутствии команды поведение текущего цикла не изменяется.

## Поток данных

`run_cycle()` → `poll_auth_command()` → проверка chat ID и команды → `reset_authentication()` → сброс приглашения → `_ensure_session()` → `ask_ready()` → существующий login-flow.

## Обработка ошибок

- Updates от чужих чатов игнорируются.
- Текст `/auth something` не считается командой.
- Ошибка Telegram API распространяется по существующему пути обработки ошибок основного loop.
- Сброс отсутствующего `storage_state.json` безопасен.
- Текущая защита от повторной отправки приглашения сохраняется для обычных циклов.

## Тестирование и критерии приёмки

- `/auth` из настроенного чата распознаётся и возвращает `True`.
- `/auth` из другого чата и `/auth extra` игнорируются.
- После обработки команды приглашение отправляется повторно с точным текстом и кнопкой «Готов».
- Сохранённый state-файл удаляется, cookies очищаются, затем запускается обычная авторизация.
- Без `/auth` существующие тесты авторизации и основного loop проходят без изменения поведения.
- Полный тестовый набор проекта проходит.

## Ограничения

- Не добавлять отдельный поток или отдельный процесс для Telegram polling.
- Не менять существующий текст приглашения.
- Не трогать `.env`, секреты, пользовательские данные и несвязанные файлы.
