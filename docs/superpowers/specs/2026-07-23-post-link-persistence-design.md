# Устойчивое сохранение ссылки на пост

## Цель

Новые комментарии, собранные из Dzen Studio, должны получать ссылку на пост,
сохраняться в `comments.post_url` и отображаться в админке как безопасная ссылка
на публикацию.

## Причина

Сейчас CSS-селектор ссылки требует `href`, начинающийся строго с `/a/`. Если Dzen
отдаёт тот же постовый `<a>` с абсолютным или иным допустимым форматом URL,
селектор не находит его, бот сохраняет пустое значение, а админка показывает
заглушку.

## Решение

- Ограничить поиск ссылкой внутри постового контейнера, но убрать из селектора
  жёсткое требование к началу `href`.
- Нормализовать относительные Dzen-ссылки при создании `Comment`, сохранив
  исходный `href` для синтетического ID и поиска узла ответа.
- Оставить проверку безопасного URL в админке: наружу разрешаются только HTTPS
  URL хоста `dzen.ru`/`www.dzen.ru` с путём `/a/...`.
- Добавить тесты для относительного и абсолютного `href`, а также проверить
  прохождение значения через `Comment` и репозиторий; существующую работу
  админки не менять.

## Критерии готовности

1. Постовый `<a>` с относительным `/a/post-id` даёт `Comment.post_url` равный
   `https://dzen.ru/a/post-id`.
2. Постовый `<a>` с абсолютным `https://dzen.ru/a/post-id` даёт тот же URL.
3. `upsert_comment` сохраняет значение в `comments.post_url` и обновляет его
   при повторном извлечении комментария.
4. Админка отображает сохранённый безопасный URL как ссылку в новой вкладке.
5. Если URL не найден или не проходит проверку безопасности, ссылка не
   отображается как кликабельная.

## Объём

Изменяются только селектор извлечения постовой ссылки и связанные тесты.
Миграция не требуется: колонка `comments.post_url` уже существует. Исторические
строки с `NULL` не backfill-ятся автоматически, потому что надёжно восстановить
их URL из текущей строки комментария нельзя.
